共享文件夹审计

企业内有足够的预算，共享文件夹的监控方案可以采用卓豪的方案，本着公司一毛不拔的原则，直接 PASS 掉。那么如何监控共享文件夹的访问和使用情况呢 ？

日志审计

我们可从审计 windows 用户的日志入手，需要涉及到日志转发，并且在筛选日志的时候非常麻烦，往往一个删除或添加操作会处触发多个事件ID，所以这里也是非常不推荐

EaseFilterSDK

从驱动层面入手，可以直接使用 EaseFilterSDK，当然该驱动在使用的时候是收费，非常成熟。除此之外，也可以自己从驱动层面来实现。当然，本文暂时不提供如何实现，后续再专门写一篇文章

Directory Monitor

Directory Monitor 也是要付费的，这里毫无疑问采用 crack 版本，我们正常安装 Directory Monitor，安装后，解压 Patch-dm2108x.rar 压缩包，然后把解压后的 exe 放到 Directory Monitor 安装路径里面，以管理员身份执行即可 crack

正常打开软件显示如下所示：

但在企业中实际使用过后，发现在监控远程共享路径的时候总是会出现一些问题，有时仅能够获取文件夹的添加、删除等，但是无法获取到是哪个用户操作的（可能是环境设置有问题），这样就有点鸡肋。

FileAduit

FileAduit 相对于 Directory Monitor 更加成熟，当然毫无疑问肯定也是收费的，这里也采用 FileAudit 6.5.0.28 的 Crack 版，安装 FileAudit 6.5.0.28 版本后，只需替换 Assemblies 文件夹下的 FileAuditAPI.dll 和 FileAuditLibrary.dll 文件即可

当然，我们在替换的时候可能会出现一些问题，如：无法替换 FileAuditAPI.dll 和 FileAuditLibrary.dll 文件的问题
